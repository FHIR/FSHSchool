{{< /blocks/cover >}} {{% blocks/lead color="primary" %}} FHIR Shorthand (FSH) is a domain-specific language for defining the contents of FHIR Resources and Implementation Guides. Explore FSH School to get the knowledge you need to get started using FSH. {{% /blocks/lead %}} {{< blocks/section color="white">}} {{% blocks/feature icon="far fa-file-alt" title="Declarative" %}} FSH allows you to define FHIR profiles in a simple, declarative language. No more large JSON files or spreadsheets, just human-readable text files which compile to the FHIR artifacts you need. {{% /blocks/feature %}} {{% blocks/feature icon="fab fa-github" title="Source control ready" %}} Unlike other tools for profiling FHIR, FSH works seamlessly with source control tools like GitHub. Maintain a history of changes, track contributions, and maintain agility by using FSH with source control. {{% /blocks/feature %}} {{% blocks/feature icon="fas fa-fire" title="State of the art" %}} The FSH Specification is continuously being improved to give users a better experience.
